## Novedades principales

| Área                            | Qué ganaste                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------- |
| **Actualización de instancias** | Runbook [`core/upgrade-guide.md`](../../core/upgrade-guide.md) y prompt `upgrade-kit` |
| **Trazabilidad**                | `kit.installed_version` en `sdd.config.yaml` y `UPGRADE-LOG.md` por proyecto          |
| **Validación**                  | `validate-sdd` advierte si la versión registrada difiere del kit en disco             |
| **Perfil mantenedores**         | `profiles/sdd-kit/` (SDD-001) para dogfooding del repositorio                         |
| **Ciclo SDD**                   | Momentos semánticos vs prompts; `verify-implementation` antes de push/PR (SDD-003)    |
| **Prompts**                     | `build-spec` unifica aprobación e implementación; deprecación suave de prompts viejos |

---

## Actualizar tu proyecto consumidor

1. Lee [`core/upgrade-guide.md`](../../core/upgrade-guide.md).
2. Ejecuta `python sdd-kit/cli/sdd.py prompt show upgrade-kit --full`.
3. Actualiza submodule, merge de instancia, `UPGRADE-LOG.md` y `kit.installed_version`.
4. Reinstala adaptadores de agente si cambió el manifest (`install-agents.py`).
